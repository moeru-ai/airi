# 通用配置

本章节介绍适用于所有服务商（Providers）的全局配置。这些设置直接影响 AIRI 的“思考”速度、对话的连贯性以及语音合成（TTS）的自然度。

## 1. 网络代理 (Proxy)

由于部分服务商（如 OpenAI, Anthropic, OpenRouter）在特定地区无法直连，你可能需要配置全局代理。

- **配置方式**：在 [设置] -> [网络] 中填写。
- **推荐格式**：`http://127.0.0.1:7890` 或 `socks5://127.0.0.1:7890`。

::: tip 为什么配置了代理还是没反应？
请确保你的代理软件开启了“允许局域网连接”或“系统代理”模式。如果是本地部署的开发环境，请检查是否触发了环境变量中的 `HTTPS_PROXY`。
:::

## 2. 断句处理 (Sentence Splitting)

这是优化**语音对话延迟**的核心配置。AIRI 采用流式（Streaming）输入，这意味着她不需要等整句话生成完就开始读。

### 核心参数说明

| 参数名 | 建议值 | 说明 |
| :--- | :--- | :--- |
| **最小断句长度** | 15 - 20 | 较短的值会让第一声出的很快，但可能导致语调生硬。 |
| **强制截断符号** | `，。！？；\n` | 遇到这些符号时，AIRI 会尝试将其作为一段话发送给 TTS。 |
| **合并策略** | 开启 | 自动合并过短的句子，避免 TTS 出现明显的机械停顿感。 |

### 体验建议
- **想要极致响应速度？** 调小最小断句长度（如 10），适合性能强大的 ASR/TTS 服务。
- **想要语气更自然？** 调大最小断句长度（如 30），让 AI 读出更完整的长句，虽然首字延迟（TTFT）会稍微增加。

## 3. 特定文本保留策略 (Preservation Rules)

在对话过程中，LLM 可能会输出一些不适合“读出来”的内容（如 Markdown 标签、颜文字、代码块）。

### 如何配置
在配置文件或设置界面的 [文本处理] 模块中：

- **屏蔽过滤**：自动过滤 `(*/ω＼*)`、`[表情]` 等字符，避免 TTS 尝试去“朗读”乱码。
- **特定词汇翻译**：如果你希望将特定的英文简称强制读成中文（或反之），可以在这里设置替换规则。

::: warning 注意
如果你正在使用 Live2D 模型并开启了口型同步，过滤掉过多的标点符号可能会导致口型动画与声音不同步。
:::

## 4. 性能调优 (Performance Tips)

### 如何平衡“大脑”与“嘴巴”
1. **LLM 选型**：在配置 [大脑] 时，开启 **Stream (流式输出)** 是实现流畅对话的前提。
2. **TTS 预加载**：如果你发现每一句话之间都有明显的“卡顿”，请检查你的 TTS 服务商是否支持并发请求。
3. **上下文限制**：建议将上下文长度限制在 10-15 轮以内，过长的上下文会导致 API 响应变慢，且增加 Token 消耗。

## 常见问题 (FAQ)

**Q: 为什么 AI 还没说完，下一句就叠上来了？**
A: 请检查“音频缓冲”设置。如果 TTS 合成速度极快而播放器缓冲区较小，可能会出现重叠。

**Q: 为什么配置了断句，但 AI 还是憋了很久才说话？**
A: 这通常不是断句的问题，而是 LLM 响应过慢。请尝试在 `llm.md` 中更换更轻量的模型（如 `gpt-4o-mini` 或 `deepseek-v3`）。
