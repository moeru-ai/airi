# Ollama (本地模型)

Ollama 是一个开源的本地大模型运行框架，支持在 macOS、Linux 和 Windows 上快速部署 Llama 3、Gwen 2 等主流模型。

如果你非常在意**数据隐私**，或者希望在**离线环境**下使用 AIRI，Ollama 是最理想的选择。

## 第一步：安装 Ollama

1. 访问 [Ollama 官网](https://ollama.com/) 下载并安装对应系统的版本。
2. 安装完成后，在终端（Terminal/PowerShell）输入以下命令验证是否安装成功：```ollama --version```

## 第二步：下载并运行模型
AIRI 建议使用推理能力较强且对中文支持良好的模型。

1. 推荐模型：

    Qwen2-7B (阿里通义千问)：中文表现极其出色，逻辑清晰。

    Llama3-8B (Meta)：综合实力最强，社区生态丰富。

2. 在终端执行以下命令下载并启动模型：
    ```
    ollama run qwen2
    ```

    *如果你使用其他模型，将 "qwen2" 替换成相应的模型代码。*

    *初次下载需要一定时间，具体取决于你的网络环境。*


## 第三步：在 AIRI 中配置
在 AIRI 的配置文件或设置界面中，将“意识接入 (LLM)”切换为 Ollama。
| 配置项 | 推荐值 | 说明 |
| :--- | :--- | :--- |
| **API 地址** | `http://localhost:11434/v1` | Ollama 默认的本地 API 端口 |
| **思考模式** | `自动（提供商决定）` | 控制 Ollama 思考行为。GPT-OSS 仅支持低/中/高级别。 |
| **HTTP 请求头** | 无 | 如果在请求 Ollama 时需要额外 HTTP 请求头，在此添加相应的键和值。常见的HTTP请求头包括 API Key （如果你使用的是云端 Ollama）|


## 硬件建议与优化
由于模型在本地运行，性能高度依赖你的硬件配置：

 - 内存 (RAM)：运行 7B/8B 规格的模型建议至少拥有 16GB 内存。

 - 显存 (VRAM)：建议 4GB 以上 。
 - 神经网络处理器 (NPU): 如果你的处理器有 NPU 核心，可以显著降低 CPU 和 GPU 的负载。
 ::: tip 提示
请注意标准版 Ollama 主要使用 CPU/GPU。如果你想使用 NPU 加速，请使用 OpenVINO 版 Ollama。
:::

- 并发处理：如果 AIRI 响应变慢，请检查任务管理器中 CPU/GPU 的占用情况，尝试关闭其他高负载应用。

::: tip 提示
你可以通过修改系统环境变量 OLLAMA_HOST 来让局域网内的其他设备也能访问你的本地模型服务。
:::


---

### 使用 Ollama 的优点

1.  **隐私保护**：如果你不想将你的对话上传到云提供商，你可以本地部署模型并用 Ollama 调用。
2.  **低成本**：不需要付费，非常适合开发者在本地反复调试模型。
3.  **硬件联动**：在带有性能较强的 CPU/GPU 甚至 NPU 的机器上运行大模型应该非常顺滑，不会受到网络限制，可以随时调用。
