title: 桌面端开发者工具
description: 理解和使用 AIRI 桌面端「系统 → 开发者」中的诊断与验证工具
---

桌面版的「系统 → 开发者」是一组为开发、排障和验证实验功能准备的工具。它们不会改善日常聊天或角色体验，也不需要在首次安装后配置；只有在复现问题、开发功能，或维护者要求收集诊断信息时才需要打开。

本页只说明桌面版。网页端虽也有开发相关页面，但可用的功能和运行环境不同。

::: warning 使用前先确认目的
部分工具会读取屏幕、使用麦克风、注册全局快捷键、打开额外窗口，或显示原始网络及插件数据。测试结束后请关闭不再使用的窗口和捕获流；不要把包含 API Key、聊天内容、屏幕画面或 WebSocket 数据的截图直接公开。
:::

## 如何进入与选择工具

在桌面端打开「设置 → 系统 → 开发者」。页面上半部分是快捷操作和渲染开关；下半部分则是不同的诊断页面。

选择工具前，可以先按问题类型判断：

| 想确认什么 | 优先使用 |
| --- | --- |
| 页面报错、元素样式或网络请求 | 打开开发者工具 |
| 页面卡顿、切换动画异常 | Lag 可视化、动画开关 |
| 按键、鼠标、显示器或全局快捷键 | useMagicKeys、鼠标／显示器工具、Global Shortcut |
| 聊天上下文、WebSocket 或实时转写 | Context Flow、WebSocket Inspector、Aliyun Real-time Transcriber |
| 插件发现、加载或卸载 | Plugin Host Debug |
| 更新失败或更新源异常 | Updater |
| 屏幕共享、视觉输入或权限问题 | Screen Capture、Vision Capture |

## 快捷操作与渲染诊断

### 打开开发者工具

点击「打开」会打开 Electron 内置的浏览器开发者工具。它适合查看控制台错误、网络请求、DOM 结构和性能记录，是定位界面问题的首选入口。

如果问题可以稳定复现，建议先清空控制台，重复一次操作，再保存相关的错误信息；提交 issue 或 PR 时应移除敏感内容。

### Markdown 压力测试

此工具会在独立窗口渲染高负载 Markdown，用来检查长文、代码块、表格和样式在较大内容量下的表现。它适用于排查 Markdown 渲染卡顿、滚动异常或主题样式错误，不会修改你的文档或聊天记录。

### IO Tracer

IO Tracer 会打开输入输出事件的诊断窗口。它用于观察调试期间的事件流向，适合检查一次操作是否产生了预期的输入、输出或状态变化。事件内容可能包含上下文数据，因此仅在需要时开启，并避免共享完整记录。

### Lag 可视化

Lag 可视化用于查看界面性能和帧时间。当窗口移动、舞台切换、角色渲染或动画出现明显卡顿时，可先用它确认问题是否与渲染负载有关。它帮助定位问题，不会自动优化性能；记录复现步骤、设备环境和帧时间变化会比单独的截图更有帮助。

### 舞台切换与页面过场动画

「是否开启舞台切换动画」控制舞台切换时的总体动画；关闭后可减少动画对测试的干扰。「是否使用页面特定过场动画」控制各页面自己的过场效果，并且在总体舞台切换动画关闭时不可用。

排查闪烁、页面未卸载或切换卡顿时，可以分别测试两种状态。完成测试后，建议恢复你平时使用的设置。

## 输入、鼠标与显示器

### useMagicKeys

这是键盘快捷键状态的调试页，用于确认应用是否正确接收修饰键和按键状态。当前没有面向普通用户的配置项；若只想更改 AIRI 的 Spotlight 快捷键，请使用「系统 → 窗口快捷方式」。

### useElectronWindowMouse、Displays 与 Relative Mouse

这三个工具分别帮助检查不同坐标系中的鼠标位置：

- **useElectronWindowMouse**：查看鼠标在所有屏幕组成的桌面坐标系中的位置；
- **Displays**：查看当前连接的显示器及鼠标所在位置，适合多显示器、缩放比例或外接屏幕问题；
- **Relative Mouse**：查看鼠标相对于 AIRI 窗口的位置，适合检查窗口内部命中区域和拖拽行为。

报告窗口跟随、点击偏移或多屏定位问题时，请一并说明显示器排列、缩放比例、主显示器和复现步骤。

### Widgets Calling

Widgets Calling 用于创建覆盖层小组件并验证传入的组件参数。它主要服务于桌面浮层和组件调用开发；如果只是使用 AIRI，不需要在这里进行任何操作。

### Beat Sync Visualizer

Beat Sync Visualizer 会绘制同步音律的 V-motion 目标、轨迹以及 Y/Z 标量变化。它用于确认声音或节拍驱动的角色运动是否连续、稳定，并协助比较不同输入下的运动数据。

## 聊天、实时服务与网络

### Context Flow

Context Flow 用于检查进入聊天管道的上下文更新，以及发往服务端的聊天流事件。它很适合验证插件、VS Code 等外部来源提供的上下文是否按预期抵达 AIRI。

推荐的排查方式是：先打开工具，再执行一次最小复现操作，最后根据时间顺序检查输入上下文与输出事件是否成对出现。上下文可能包含正在编辑的文件名、聊天内容或其他私密信息，分享日志前务必脱敏。

### WebSocket Inspector

WebSocket Inspector 显示原始 WebSocket 流量。它适用于连接建立失败、事件未送达、消息格式不符合预期等问题。因为它展示的是底层协议数据，通常应只截取与问题直接相关的少量帧，并删除令牌、用户内容和地址信息。

### Aliyun Real-time Transcriber

这个页面将麦克风音频流发送到阿里云 NLS，并显示实时转写结果。它用于验证实时语音识别链路，包括麦克风输入、凭据、网络和转写输出。开始前请确认已选择正确输入设备，并只在你有权录音的环境中测试。

## 插件、更新与系统功能

### Plugin Host Debug

Plugin Host Debug 用于检查插件是否被发现、启用和加载，并可控制插件的加载与卸载生命周期。插件未生效时，依次检查：插件是否被发现、是否已启用、加载过程中是否出现错误，以及卸载后是否仍残留事件或界面状态。

不要为了“修复”一个普通使用问题随意反复加载和卸载插件；先记录状态和错误信息，再进行最小改动测试，结果会更容易复现。

### Updater

Updater 展示当前版本、平台、架构、更新通道、更新源、日志位置和更新状态，也可以手动检查、下载和安装更新。它主要用于排查更新失败、错误的更新源或特定平台的安装问题。

日常升级优先使用「关于」窗口；在开发者工具中覆盖更新源前，请确认自己了解该来源的可信性和影响范围。

## 屏幕与视觉捕捉

### Screen Capture

Screen Capture 可捕获应用窗口、整块显示器或系统暴露的其他设备，并创建视频或音频流。它主要用于验证屏幕共享和捕获链路。

首次使用时，系统会要求授权屏幕录制权限。若 macOS 没有列出 AIRI，可在「系统设置 → 隐私与安全性 → 屏幕与系统音频录制」中手动添加或启用 AIRI，然后重启应用再测试。Windows 和 Linux 的可用设备与权限提示会随系统、桌面环境和浏览器／Electron 版本而变化。

工具中的 `applications` 用于选择某个应用窗口，`displays` 用于选择整块显示器，`devices` 列出系统报告的可捕获设备；连接或断开显示器、打开新窗口、修改权限后，可使用 `refetch` 刷新列表。停止捕获后应确认预览已关闭，避免继续占用权限或资源。

### Vision Capture

Vision Capture 会捕获屏幕帧，并显示提交给视觉处理流程的输出载荷。它用于验证视觉输入是否成功取得、帧数据是否正确，以及下游视觉功能是否获得预期内容。它不是一个持久开启的全局开关：仅配置视觉服务商和模型时，不需要打开此页。

使用屏幕视觉时，依次完成以下步骤：

1. 在「设置 → 服务商 → 视觉」中选好视觉服务商和支持图像输入的模型；
2. 打开「系统 → 开发者 → Vision Capture」，完成系统的屏幕录制授权；
3. 选择窗口或显示器，再点击「Start ticker」开始捕获和分析；
4. 仅在希望将识别结果加入 AIRI 对话上下文时，开启「Publish to character」；
5. 测试完成后点击「Stop ticker」。离开此页面也会停止捕获循环。

若页面一直停在权限提示，先在操作系统中完成授权、完全退出并重新启动 AIRI，再重新打开此工具；不要把包含个人桌面、通知或其他应用内容的捕获结果上传到公开位置。

## 全局快捷键

### Global Shortcut

Global Shortcut 用于注册、注销和观察系统级快捷键事件。它与「系统 → 窗口快捷方式」中给普通用户设置的 Spotlight 快捷键不同：前者用于开发与验证，后者用于日常使用。

测试前选择一个不与操作系统或常用应用冲突的组合键。若注册失败，先检查该组合键是否已经被占用；测试结束后注销快捷键，避免它继续在后台拦截按键。
