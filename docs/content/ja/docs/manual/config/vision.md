---
title: ビジョンを構成する
description: AIRIのビジョンモジュールの画像理解サービスプロバイダーとモデルの選択
---

ビジョン モジュールは、画面またはカメラ画像を理解するための画像入力をサポートするモデルにキャプチャされた画像を送信します。 AIRI は、ビジュアル API 認証情報の個別のセットを維持しません。各チャット プロバイダーに対応するビジュアル設定項目を提供し、プロバイダーのフィールドを再利用します。

::: info ビジュアル設定とチャット設定が共有されるのはなぜですか?
多くの場合、同じサービス プロバイダーがチャット モデルと画像理解モデルの両方を提供します。認証情報を再利用すると、構成の重複が減ります。テキストのみのモデルでは視覚的なタスクを実行できないため、画像入力を明示的にサポートするモデルを選択する必要があります。
:::

::: 警告 スクリーンビジョンを使用する前に、ビジョンキャプチャを開始する必要があります
ビジョン サービス プロバイダーとモデルのみを構成する場合、このツールを有効にする必要はありません。

AIRI に画面またはウィンドウを分析させるには、「システム → 開発者 → ビジョン キャプチャ」に移動します。画面録画の許可を与え、キャプチャしたいウィンドウまたはディスプレイを選択し、「ティッカーの開始」をクリックします。認識結果をAIRIダイアログに提供したい場合は「キャラクターに公開」をオンにします。

Vision Capture は、現在のデスクトップ デバッグ/開発ワークフローです。ページを離れるとキャプチャ ループが停止します。完全な手順については、[デスクトップ開発者ツール](../../contributing/desktop-developer-tools#vision-capture) を参照してください。
:::

## ステップ 1: ビジュアル サービス プロバイダーを選択する

1. **[設定] → [サービス プロバイダー] → [ビジョン]** を開きます。
2. 構成済み、または構成する予定のチャット サービス プロバイダーを選択します。
3. サービス プロバイダー カードに従って資格情報を入力します。フィールドは、API キー、ベース URL、Azure リソース情報、Amazon Bedrock リージョンなどのチャット プロバイダーのバージョンと一致します。

ビジュアル サービス プロバイダーとチャット サービス プロバイダーの間には 1 対 1 の対応があります。まずサイドバーの「サービス プロバイダー → チャット サービス プロバイダー」から対応するサービス プロバイダーの構成を完了し、次にビジュアル ページで同じ資格情報を使用します。ビジュアル ポータルが利用できるからといって、すべてのモデルが画像を読み取れるわけではありません。画像入力を明示的にサポートするモデルは、引き続きモデル リストで選択する必要があります。

::: 警告 画像と認証情報のセキュリティ
視覚分析により、選択したサービス プロバイダーに映像が送信されます。 API キー、パスワード、個人情報、または未承認のコンテンツを含む映像を撮影しないでください。また、クラウド サービスの認証情報をリポジトリに送信したり、スクリーンショットを撮ったり、他の人に送信したりしてはなりません。
:::

## ステップ 2: ビジュアル モデルを選択する

1. **[設定] → [ビジョン]** を開きます。
2. 構成したばかりのサービスプロバイダーを選択します。
3. モデル リストから画像またはビジュアル入力をサポートするモデルを選択します。
4. 必要なビジュアル機能を有効にし、ページの指示に従って画像ソースまたはキャプチャ方法を選択します。必要に応じて、「キャプチャ間隔」であるキャプチャ間隔を設定して、キャプチャ時間間隔を調整できます。

## ステップ 3: 構成の検証

1. 機密情報が含まれていないテスト画面を使用します。
2. 視覚的な分析をトリガーします。
3. AIRI が画面の説明または対応するコンテキストを返した場合、サービス プロバイダー、モデル、および画面入力が正常に設定されたことを意味します。

## ローカルビジュアルモデル

Ollama と LM Studio は、ローカルのビジュアル サービス プロバイダーとして利用できます。まず、画像入力をサポートするモデルをローカルで実行し、そのサービス アドレスが AIRI からアクセスできることを確認します。次に、ビジュアル サービス プロバイダー ページで対応するベース URL を入力するか保持し、モデル リストからビジュアル モデルを選択します。

## トラブルシューティング

|現象|優先順位のチェック|
| --- | --- |
|サービスプロバイダーは保存できません|API キー、Azure リソース名、Bedrock リージョンなど、チャット バージョンと同じ資格情報フィールドが入力されます。|
|モデルは画像を解析できません|モデルが画像入力を明示的にサポートしているかどうか。サービスプロバイダーが提供するビジュアルモデルに切り替えます。|
|ローカルモデルに到達できません|ローカル サービスが実行されているかどうか、ベース URL、ポート、CORS、および LAN アクセスの設定。|
|リクエストが拒否されたか、割り当てが不十分でした。|サービス プロバイダーのアカウント権限、モデルの利用可能なエリア、クォータ、およびネットワーク接続。|
