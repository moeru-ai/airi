このページでは、AIRI のプロバイダー構成がどのように機能するかについて説明します。特定のサービスプロバイダーのAPIアドレス、口座開設方法、機種の選択については、各サービスプロバイダーのページをご覧ください。

## 構成プロセス

1. **[設定] → [サービス プロバイダー]** を開き、**チャット**、**音声合成**、または **音声認識**の機能カテゴリを選択します。
2. サービスプロバイダーを選択し、ページで必要な資格情報を入力します。
3. 必要に応じて、詳細設定を展開し、サービス プロバイダーが提供するベース URL またはその他のパラメータを入力します。
4. AIRI が検証を完了するまで待ちます。検証に合格したら、サービスプロバイダーと、対応する機能モジュールのモデルまたはサウンドを選択します。

::: 警告 認証情報のセキュリティ
資格情報とプロバイダー設定は、現在のデバイスのローカル設定に保存されます。スクリーンショット、ログ、問題、チャット ログなどで API キー、AccessKey Secret などの認証情報を決して開示しないでください。
:::

## 共通フィールド

|分野|意味|提案を記入してください|
| --- | --- | --- |
| API Key |サービスプロバイダーが発行するアクセストークン|引用符やスペースを追加せずに、完全なキーを直接貼り付けます。|
| Base URL |サービスプロバイダーAPIのルートアドレス|サービス プロバイダーのドキュメントで要求されている場合にのみ変更してください。カスタムアドレスを使用する場合は、完全な `https://` または `http://` アドレスを入力します。|
|モデル|チャット、音声、または認識に使用されるモデル ID|AIRI のリストから選択することを優先します。リストの読み込みに失敗した場合は、サービスプロバイダーのドキュメントに従って手動でリストを入力してください。|
|音色|TTSが読み上げる際に使用される音声ID|まずモデルを選択し、次にそのモデルがサポートするサウンドを選択します。|
|エリア|一部のクラウド サービスはアクセス ノードを決定するために使用されます|サービス プロバイダー コンソールのプロジェクトまたはサービス領域と一致している必要があります。|

## 検証結果

チャット プロバイダーは通常、ネットワーク接続、モデルのリスト、および簡単なチャット リクエストをチェックします。最後のアイテムは、少量のサービス プロバイダー クレジットを消費する可能性があります。音声サービスプロバイダーおよび音声認識サービスプロバイダーは、対応するモジュールのテスト領域で実際の再生または認識結果を検証します。

検証が失敗した場合は、次の順序でトラブルシューティングを行ってください。

1. アカウントが対応するサービスをアクティブ化しており、利用可能なクレジットがあることを確認します。
2. API キーを再度コピーし、余分なスペースや改行がコピーされていないか確認します。
3. ベース URL をサービス プロバイダーのデフォルト値に復元するか、サービス プロバイダーの公式ドキュメントで逐語的に確認します。
4. ネットワーク、プロキシ、およびファイアウォールがサービス プロバイダーへのアクセスを許可していることを確認します。
5. プロバイダーによって明示的にサポートされているモデルを選択します。表示名をモデル ID として使用しないでください。
