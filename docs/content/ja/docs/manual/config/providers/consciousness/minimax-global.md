このページは、MiniMax 海外プラットフォームで作成された API キーに適用されます。設定が完了すると、AIRI は MiniMax Global in Affairs によって提供されるチャット モデルを使用できるようになります。

::: info MiniMax Globalを選ぶ理由?
MiniMax 海外プラットフォームで API キーを作成する場合、または海外のトークン プランを使用する場合は、MiniMax Global を選択する必要があります。中国本土のプラットフォームで作成されたキーには [MiniMax (中国本土)](./minimax.md) を使用してください。 2 つのプラットフォームの API キー、請求、およびベース URL を混合することはできません。
:::

## ステップ 1: API キーを取得する

1. [MiniMax Global Platform](https://platform.minimax.io/)。)を開いてログインします。
2. **API キー**で従量課金制 API キーを作成します。トークン プランを使用している場合は、対応するサブスクリプション ページで専用のキーを取得してください。
3. キーをコピーし、安全な場所に保管します。

::: warning API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを撮ったり、他の人に送信したりしないでください。キーが侵害された場合は、直ちにそのキーを取り消し、MiniMax Global プラットフォーム上に新しいキーを作成します。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [チャット] → [MiniMax Global]** を開きます。
2. API キーを基本設定に貼り付けます。
3. デフォルトのベース URL `https://api.minimax.io/v1/` をそのまま使用します。

## ステップ 3: 構成を確認する

1. **Ping API**: このボタンをクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。
2. **モデルの選択**: テストが成功したら、ここをクリックして使用する特定のモデルを選択します。
3. **[設定] → [意識]** で MiniMax Global とモデルを選択し、短いメッセージを送信して AIRI が応答できることを確認します。

## トラブルシューティング

API への ping が失敗した場合は、API キーが海外のプラットフォームからのものであること、ベース URL が `https://api.minimax.io/v1/` であることを確認し、アカウント制限とネットワーク接続を確認してください。 401 が発生する場合の一般的な理由は、中国本土と海外のプラットフォームのキーまたはアドレスが混在していることです。モデル リストのロードに失敗した場合は、MiniMax Global によって提供される正確なモデル ID を [認識] ページに手動で入力できます。
