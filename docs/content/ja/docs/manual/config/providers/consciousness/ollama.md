---
title: オラマ（現地モデル）
description: ローカル Ollama サービスを使用した AIRI のチャット モデルの設定
---

Ollama は、オープンソースのローカル大規模モデル実行フレームワークです。独自のデバイスでモデルを実行し、デフォルトでは API キーを必要としないローカル API を提供できます。

::: info オラマを選ぶ理由
モデルをネイティブに実行したい場合、クラウド API への依存を軽減したい場合、または会話データのローカル処理に重点を置きたい場合は、Ollama が正しい選択です。
:::

## ステップ 1: Ollama をインストールする

1. [Ollama 公式サイト](https://ollama.com/)から、ご利用のシステムに対応するバージョンをダウンロードしてインストールします。
2. ターミナル (ターミナルまたは PowerShell) で次のコマンドを実行して、インストールが成功したことを確認します。

    ```bash
    ollama --version
    ```

## ステップ 2: モデルをダウンロードして実行する

1. ターミナルで次のコマンドを実行して、モデルをダウンロードして開始します。

    ```bash
    ollama run qwen2
    ```

2. 他のモデルを使用する場合は、`qwen2` を対応するモデル ID に置き換えます。モデルの初回ダウンロードに必要な時間は、モデルのサイズとネットワーク環境によって異なります。

## ステップ 3: AIRI で設定する

1. **「設定」→「サービスプロバイダー」→「チャット」→「Ollama」**を開きます。
2. デフォルトのベース URL: `http://localhost:11434/v1/` をそのまま使用します。 Ollama が別のデバイスで実行されている場合は、デバイスがアクセスできるアドレスを入力します。
3. Ollama と、「認識」でダウンロードしたモデルを選択します。

## ステップ 4: 構成を確認する

1. **Ping API**: このボタンをクリックして、AIRI がローカル サービスに接続できるかどうかをテストします。
2. **モデルの選択**: テストが成功したら、ここをクリックしてダウンロードしたモデルを選択します。

## トラブルシューティング

接続できない場合は、まず Ollama が実行中であり、ポートがベース URL と一致していることを確認してください。 AIRI と Ollama が同じデバイス上にない場合は、AIRI デバイスからアクセスできる LAN アドレスを使用し、信頼されたネットワーク上でのみサービスを開きます。

::: warning AIRI はローカル Ollama に接続できません
Ollama が実行中であるが、AIRI がネットワーク エラーまたは CORS エラーを示している場合は、Ollama を起動したときと同じように `OLLAMA_ORIGINS` をセットアップし、AIRI のオリジンがサービスにアクセスできるようにしてから、Ollama を再起動します。問題のトラブルシューティングのために、ローカル サービスをパブリック ネットワークに直接公開しないでください。
:::
