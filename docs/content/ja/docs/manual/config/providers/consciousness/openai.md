公式 OpenAI アドレスを使用する場合は、**OpenAI** を選択します。サードパーティ互換のアドレスを使用する場合は、**OpenAI 互換 API** を選択します。設定が完了すると、AIRIは「認識」で対応するサービスプロバイダーのチャットモデルを利用できるようになります。

::: info OpenAI または互換性のある API を選択する理由は何ですか?
すでに OpenAI API キーを持っている場合、またはサービス プロバイダーが OpenAI 互換のチャット インターフェイスを明示的に提供している場合は、この設定方法を使用できます。 `/v1` で終わる API アドレスまたは `sk-` で始まるキーを持っているだけでは、サービスの互換性は保証されません。
:::

## ステップ 1: API キーを取得する

1. [OpenAI API Keys](https://platform.openai.com/api-keys) を開いてログインし、必要な API キーまたは認証情報を作成します。
2. API キーまたは開発者設定ページで API キーを作成します。
3. キーをコピーし、安全な場所に保管します。

::: warning API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを含めたり、他の人に送信したりしないでください。キーが侵害された場合は、すぐにそれを取り消し、プロバイダー コンソールで新しいキーを作成します。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [チャット] → [OpenAI]** または **OpenAI 互換 API** を開きます。
2. API キーを基本設定に貼り付けます。
3. OpenAI 公式サービスを使用する場合は、デフォルトのベース URL: `https://api.openai.com/v1` を維持します。互換性のあるサービスを使用する場合は、サービス プロバイダーのドキュメントで提供される API ルート アドレスを入力し、`/chat/completions` パスを追加しないでください。

## ステップ 3: 構成を確認する

1. **Ping API**: このボタンをクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。
2. **モデルの選択**: テストが成功したら、ここをクリックして使用する特定のモデルを選択します。

## トラブルシューティング

API への ping が失敗した場合は、API キー、アカウント制限、ネットワーク接続を確認してください。互換性のあるサービスを使用する場合は、そのサービスが OpenAI Chat Completions API を明示的にサポートしていることを確認し、ベース URL がサービス プロバイダーのドキュメントで指定されているルート アドレスであることを確認してください。
