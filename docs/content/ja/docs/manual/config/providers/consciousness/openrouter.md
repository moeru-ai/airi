OpenRouter は、アグリゲーション API サービス プロバイダーです。このページでの設定を完了すると、AIRI は、Awareness で OpenRouter が提供するチャット モデルを使用できるようになります。

::: info OpenRouter を選ぶ理由
1 つの API キーで AIRI の複数のモデルを試したい場合は、OpenRouter が便利な選択肢です。複数のモデルのサービスを同じソケットと請求書に集中化するため、通常、モデルを切り替えるときに複数のサービス プロバイダーを個別に構成する必要はありません。中国本土で AIRI を使用する場合は、最初に 302.AI を試すことができます。実際の可用性は、ネットワーク環境、支払い方法、サービス プロバイダーのポリシーによって異なります。
:::

## ステップ 1: API キーを取得する

1. [OpenRouter API キー](https://openrouter.ai/keys)，创建新的 API キー)を開きます。
2. キーの適切な名前、有効期間、割り当て制限を設定します。
3. キーをコピーし、安全な場所に保管します。

::: 警告 API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを含めたり、他の人に送信したりしないでください。キーが侵害された場合は、直ちにそのキーを取り消し、OpenRouter コンソールで新しいキーを作成します。
:::


## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [チャット] → [OpenRouter]** を開きます。
2. API キーを基本設定に貼り付けます。
3. デフォルトのベース URL `https://openrouter.ai/api/v1` をそのまま使用します。

## ステップ 3: 構成を確認する

1. **Ping API**: このボタンをクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。
2. **モデルの選択**: テストが成功したら、ここをクリックして、使用する特定のモデル (**google/gemini-pro-1.5** など) を選択します。

## トラブルシューティング

API への ping が失敗した場合は、API キー、アカウント制限、ネットワーク接続を確認してください。モデル リストのロードに失敗した場合は、OpenRouter によって提供される正確なモデル ID を [認識] ページに手動で入力できます。
