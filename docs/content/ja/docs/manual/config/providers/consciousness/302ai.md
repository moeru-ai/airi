302.AI は、アグリゲーション API サービス プロバイダーです。このページでの設定を完了すると、AIRI は「認識」で 302.AI が提供するチャット モデルを使用できるようになります。

::: info 302.AIを選ぶ理由?
主に中国本土で AIRI を使用する場合は、まず 302.AI を試してください。集約された API を提供するため、1 つの API キーで複数のモデルを簡単に試すことができます。実際の可用性は、ネットワーク環境、支払い方法、サービス プロバイダーのポリシーによって異なります。
:::

## ステップ 1: API キーを取得する

1. [302.AIコンソール](https://302.ai/) を開いてログインし、必要な API キーまたは認証情報を作成します。
2. コンソールで API キーを作成します。
3. キーをコピーし、安全な場所に保管します。

::: warning API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを含めたり、他の人に送信したりしないでください。キーが侵害された場合は、すぐにそれを取り消し、302.AI コンソールで新しいキーを作成します。
:::

## ステップ 2: AIRI で設定する

1. **「設定」→「サービスプロバイダー」→「チャット」→「302.AI」**を開きます。
2. API キーを基本設定に貼り付けます。
3. デフォルトのベース URL `https://api.302.ai/v1` をそのまま使用します。

## ステップ 3: 構成を確認する

1. **Ping API**: このボタンをクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。
2. **モデルの選択**: テストが成功したら、ここをクリックして使用する特定のモデルを選択します。

## トラブルシューティング

API への ping が失敗した場合は、API キー、アカウント残高、ネットワーク接続を確認してください。モデル リストのロードに失敗した場合は、302.AI によって提供される正確なモデル ID を [認識] ページに手動で入力できます。
