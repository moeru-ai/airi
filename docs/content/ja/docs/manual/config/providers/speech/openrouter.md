---
title: OpenRouter（TTS）
description: AIRI で OpenRouter を音声合成サービス プロバイダーとして設定する
---

OpenRouter は、アグリゲーション API サービス プロバイダーです。設定が完了したら、「Voice」でOpenRouterが提供する音声モデルと音色を選択します。

::: info OpenRouter Voice を選択する理由
同じ OpenRouter アカウント内で複数のモデルと音声機能を管理する場合は、このプロバイダーを選択してください。中国本土で AIRI を使用する場合は、最初に 302.AI を試すことができます。実際の可用性は、ネットワーク環境、支払い方法、サービス プロバイダーのポリシーによって異なります。
:::

## ステップ 1: API キーを取得する

1. [OpenRouter API キー](https://openrouter.ai/keys)，创建新的 API キー)を開きます。
2. キーの適切な名前、有効期間、割り当て制限を設定します。
3. キーをコピーし、安全な場所に保管します。

::: 警告 API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを含めたり、他の人に送信したりしないでください。キーが侵害された場合は、直ちにそのキーを取り消し、OpenRouter コンソールで新しいキーを作成します。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [音声合成] → [OpenRouter]** を開きます。
2. API キーを基本設定に貼り付けます。
3. デフォルトのベース URL `https://openrouter.ai/api/v1/` をそのまま使用します。

## ステップ 3: 構成を確認する

1. 「音声」で設定したサービスプロバイダー、モデル、トーンを選択します。
2. テストテキストを入力し、「テスト」をクリックします。
3. 音声が正常に再生できれば、設定は成功です。エラーが表示された場合は、エラーメッセージに基づいて認証情報とモデルを確認してください。

## トラブルシューティング

音が出ない場合は、選択した機種が音声出力が可能であることを確認し、アカウント残高やネットワーク接続を確認してください。
