---
title: ComfyUI（芸術作品）
description: AIRI でのネイティブ ComfyUI ワークフローの接続
---

ComfyUI を使用すると、AIRI はローカル マシンまたは信頼できる LAN からイメージ生成ワークフローを使用できます。設定が完了したら、**設定 → アート** で ComfyUI を選択し、保存されたワークフローを使用して AIRI に画像を生成させることができます。

::: info ComfyUI を選ぶ理由
独自にインストールされたモデル、ノード、ワークフローを使用し、画像生成をローカル環境に残しておきたい場合は、ComfyUI が AIRI のネイティブ アート サービス プロバイダーです。
:::

## ステップ 1: ComfyUI サービスとワークフローを準備する

1. ComfyUIを起動します。 AIRI はデフォルトで `http://localhost:8188` に接続します。
2. ComfyUI で直接実行できる画像ワークフローを用意し、その API ワークフロー JSON を ComfyUI からエクスポートします。
3. AIRI と ComfyUI が同じデバイス上にない場合は、AIRI が配置されているデバイスからアドレスにアクセスできることを確認します。

::: 警告 ローカル サービスとワークフロー セキュリティ
ComfyUI のサービス ポートを信頼できないパブリック ネットワークに公開しないでください。ワークフローをインポートする前にワークフロー内のノード、モデル パス、パラメーターを確認し、不明なソースからワークフロー JSON をインポートしないでください。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [アート] → [ComfyUI]** を開きます。
2. ComfyUI サーバーの URL を入力します。本機はデフォルトで`http://localhost:8188`を使用しています。
3. [**接続のテスト**] をクリックして、AIRI が ComfyUI サービス ステータスを読み取れることを確認します。
4. API ワークフロー JSON をワークフロー領域にアップロードし、名前を入力し、AIRI に公開する入力フィールドを選択します。
5. ワークフローを保存し、アクティブなワークフローにします。

## ステップ 3: 構成を確認する

1. **[設定] → [意識]** で、ツール呼び出し / 関数呼び出しをサポートするチャット モデルを選択します。 AIRI では、モデルによって ComfyUI 画像生成ツールを呼び出す必要があります。
2. **[設定] → [アート]** を開き、**ComfyUI** を選択します。
3. 保存したワークフローを選択し、機密情報を含まないプロンプトワードを使用して生成を開始します。
4. ComfyUI のキューまたは履歴にタスクが表示されていることを確認します。ワークフローが完了して画像が返された場合、接続、ワークフロー、チャット モデル、および公開可能なフィールドが正常に構成されたことを意味します。

## トラブルシューティング

テスト接続が失敗した場合は、ComfyUI が実行されているかどうか、サーバー URL、ポート、ネットワーク アクセスを確認してください。ブラウザーがクロスドメイン エラーを報告した場合は、ComfyUI 設定ページに表示される CORS 起動パラメーターに従ってサービスを再起動します。ワークフローが実行できない場合は、インポートしたAPI形式がJSONであること、使用するノードとモデルがComfyUIにインストールされていることを確認してください。 ComfyUI キューに新しいタスクがない場合は、現在のチャット サービス プロバイダーとモデルがサポートしているかどうかを確認し、ツール呼び出し/関数呼び出しを有効にします。テキストダイアログのみをサポートするモデルでは、生成ツールをトリガーできません。
