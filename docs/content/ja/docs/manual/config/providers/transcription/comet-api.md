---
title: CometAPI（ASR/STT）
description: AIRI で CometAPI 音声認識を設定する
---

CometAPI は、互換性のあるインターフェイスを通じて音声転写を提供します。

::: info CometAPI を選ぶ理由
CometAPI を使用してモデルと認証情報を管理した場合は、同じ API キーを AIRI で音声認識に直接再利用できます。
:::

## ステップ 1: API キーを取得する

1. [CometAPI Console](https://www.cometapi.com/console/token) を開いてログインし、必要な API キーまたは認証情報を作成します。
2. アカウントが音声転写モデルにアクセスできることを確認し、キーをコピーして適切に保管します。

::: warning API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを撮ったり、他の人に送信したりしないでください。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [音声認識] → [CometAPI Transcription]** を開き、API キーを入力します。
2. デフォルトのベース URL: `https://api.cometapi.com/v1/` をそのまま使用します。プロキシまたは互換性のあるゲートウェイを使用する場合にのみ変更してください。

## ステップ 3: 構成を確認する

1. **Ping API**: このボタンをクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。
2. **モデルの選択**: テストが成功したら、利用可能な転写モデルを選択します。次に、**設定 → 聴覚** に移動して有効にします。
3. マイクへのアクセスを許可し、短い音声入力を実行してテキストを出力できることを確認します。

## トラブルシューティング

API への ping が失敗した場合は、API キー、アカウント権限、ネットワーク接続を確認してください。テキスト結果がない場合は、AIRI がシステムのマイク許可を取得していることを確認します。
