---
title: OpenAI互換API（ASR/STT）
description: AIRI での音声認識プロバイダーとしての OpenAI または OpenAI 互換 API の設定
---

このページは、OpenAI または OpenAI 互換のトランスコーディング インターフェイスを提供するサービス プロバイダーを構成するために使用されます。設定が完了したら、ヒアリングでモデルを選択し、マイク入力をテストします。

::: info ASR/STT 互換のために OpenAI を選択する理由は何ですか?
音声認識サービス プロバイダーが OpenAI 互換の文字起こしインターフェイスを明確に提供している場合は、このページで説明されているように AIRI にアクセスできます。 `/v1` で終わる API アドレスまたは `sk-` で始まるキーを持っているだけでは、サービスの互換性は保証されません。
:::

## ステップ 1: API キーを取得する

1. 選択したサービスプロバイダーの管理コンソールにログインします。
2. API キーまたは開発者設定ページで API キーを作成します。
3. キーをコピーし、安全な場所に保管します。

::: 警告 API キーのセキュリティ
API キーをリポジトリに送信したり、スクリーンショットを含めたり、他の人に送信したりしないでください。キーが侵害された場合は、すぐにそれを取り消し、プロバイダー コンソールで新しいキーを作成します。
:::

## ステップ 2: AIRI で設定する

1. **[設定] → [サービス プロバイダー] → [音声認識] → [OpenAI 互換 API]** を開きます。
2. 使用する API キーと ASR/STT モデル ID を入力します。
3. OpenAI 公式サービスを使用する場合は、デフォルトのベース URL: `https://api.openai.com/v1/` を維持します。互換性のあるサービスを使用する場合は、サービス プロバイダーのドキュメントで提供される API ルート アドレスを入力します。

## ステップ 3: 構成を確認する

1. Hearing で設定されているサービスプロバイダーとモデルを選択し、音声入力デバイスを選択します。
2. [モニタリングの開始] をクリックし、マイクに向かって話すか、オーディオ クリップを再生します。
3. 文字起こしエリアにテキストがリアルタイムに出力できることを確認します。認識結果が不正確な場合は、感度を調整して再度テストできます。

## トラブルシューティング

テキスト結果がない場合は、まずシステムが AIRI マイクの許可を与えているかどうかを確認してください。互換性のあるサービスを使用する場合は、OpenAI 翻訳インターフェースに対応していることと、モデル ID が入力されていることを確認してください。
