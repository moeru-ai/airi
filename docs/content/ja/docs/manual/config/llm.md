---
title: チャットモデルの構成
description: AIRI のチャットプロバイダーとモデルの設定
---

チャットモデルはAIRIの「頭脳」です。このページを完了すると、AIRI はテキスト応答を生成できます。チャット機能をサポートするサービス プロバイダー アカウント、または実行中のローカル モデル サービスが必要です。

## 前提条件

※AIRIがインストールされ起動します。
※クラウドサービスプロバイダー：API Keyが作成され、チャットモデルが利用できるアカウントであることが確認されています。
* ローカル サービス: モデル サービスが開始されており、AIRI が配置されているデバイスはモデル サービスにアクセスできます。

## ステップ

1. **[設定] → [サービス プロバイダー] → [チャット]** を開き、使用するサービス プロバイダーを選択します。

どれを選択すればよいかわからない場合は、まず利用可能な詳細なガイドから選択できます: [OpenRouter](./providers/consciousness/openrouter.md)、[DeepSeek](./providers/consciousness/deepseek.md)、[OpenAI 互換 API](./providers/consciousness/openai.md)、またはローカルの [Ollama](./providers/consciousness/ollama.md)。その他のチャットサービスプロバイダーは、サイドバー「サービスプロバイダー → チャットサービスプロバイダー」からご覧いただけます。

2. API キーを入力します。サービスプロバイダーが専用の API アドレスを提供する場合は、詳細設定でベース URL を入力します。それ以外の場合は、デフォルト値をそのまま使用します。

3. [**Ping API**] をクリックして、ネットワークが接続されているかどうか、また API キーが正しく入力されているかどうかをテストします。

4. ネットワークが開いたら、[**モデルの選択**] をクリックします。 **[設定] → [認識]** を開いて、構成したばかりのチャット サービス プロバイダーとモデルを選択することもできます。

サービス プロバイダーがサポートしている場合、AIRI はモデル リストを読み込みます。リストをロードできない場合は、サービス プロバイダーの公式ドキュメントに従って、正確なモデル ID を手動で入力できます。

5. チャット インターフェイスに戻り、「こんにちは」などの短いメッセージを送信します。応答を受信すると、構成が成功したことを示します。

## トラブルシューティング

### 検証は合格しましたが、オプションのモデルはありません

まず、サービス プロバイダーがモデルのリストを許可していることを確認します。一部のサービスプロバイダーはモデルリストを提供していないか、API キーに対応する権限がありません。この場合、「認識」ページでモデル ID を手動で入力できます。モデル ID はプロバイダーのドキュメントと正確に一致する必要があります。

### 検証が失敗したか、リクエストがタイムアウトしました

API キー、ベース URL、アカウント制限、ネットワーク接続を確認します。ローカル サービスの場合は、サービスが実行中であること、およびベース URL が他のデバイスのみがアクセスを許可されているアドレスではないことを確認します。

### アイリ 返事はありません

「認識」ページでサービスプロバイダーとモデルの両方が選択されていることを確認します。プロバイダーの資格情報を保存するだけでは、自動的に有効になりません。

## 次のステップ
