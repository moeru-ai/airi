AIRI と会話できるようにするには、少なくとも 1 つのチャット サービス プロバイダーと 1 つのチャット モデルを設定する必要があります。音声合成 (TTS) により、AIRI は話すことができます。音声認識 (ASR/STT) により、マイク入力を理解できます。これら 2 つの項目はオプションですが、完全な音声対話エクスペリエンスを実現するには、これらを一緒に構成することをお勧めします。

## 最初に利用可能な最小限の構成を完了します

1. AIRI の **[設定] → [サービス プロバイダー]** を開きます。
2. **チャット** カテゴリでサービス プロバイダーを選択し、資格情報を入力して検証を完了します。
3. **[設定] → [認識]** を開き、構成したサービス プロバイダーとモデルを選択します。
4. メッセージを送信して、AIRI が返信できることを確認します。

チャットの構成が完了したら、必要に応じて音声を構成します。

* **[一般的な説明](./common.md)**: 設定プロセス、フィールドの意味、検証結果、FAQ を理解します。
* **[チャット モデルの構成](./llm.md)**: LLM を構成し、Awareness でモデルを選択します。
* **[音声入力と出力の設定](./audio.md)**: TTS と ASR/STT を設定し、「サウンド」と「聴覚」で有効にします。
* **[Configure Visual Understanding](./vision.md)**: 構成されたチャット プロバイダーで画像入力をサポートするモデルを AIRI に使用させます。
* **[Web 検索の設定](./web-search.md)**: Tavily を使用すると、必要に応じて AIRI がインターネットで最新情報を検索できるようになります。
* **サービス プロバイダー**: サイドバーから [サービス プロバイダー] メニューを展開し、[チャット]、[音声合成]、[音声認識]、または [アート作成] を押して、対応する構成ガイドを開きます。芸術的な創作にローカル ワークフローが必要な場合は ComfyUI を選択してください。クラウド生成が必要な場合は、「Replicate」または「Nano Banana」を選択します。

> [!TIP]
> 最初に AIRI が動作することを確認したいだけの場合は、最初にチャット プロバイダーを設定します。 TTS と ASR は、チャットが正常になった後に追加できるため、問題の特定が容易になります。

＃＃ 設定

サービスプロバイダーの設定が完了したら、設定で AIRI のテーマカラーを変更したり、Live2D (2D) モデルと VRM (3D) モデルを切り替えたりすることもできます。

サービス プロバイダーを設定するときは、サービス プロバイダーのドキュメントで提供されるデフォルトのアドレスとモデル名の使用を優先してください。ベース URL、モデル ID、または領域パラメーターを推測しないでください。プロバイダーによって異なります。

### 機種変更

デフォルトのモデルを別の Live2D (2D) モデルまたは VRM (Grok Companion に似た 3D モデル (モデルがある場合)) に置き換えることができます。

モデルの設定は[設定] -> [モデル]にあります。

::: tip VTuber Studio からモデルをインポートしますか?
VTuber Studio は Live2D エンジンが認識しないファイルを使用するため、Live2D モデルのレンダリングに使用するライブラリでは、VTuber Studio によってパッケージ化された ZIP ファイルの読み取りに問題が発生する可能性があります。
そのため、VTuber Studio モデルを ZIP ファイルに圧縮してインポートする場合は、以下のファイルを必ず除外してください。

-`items_pinned_to_model.json`
:::
