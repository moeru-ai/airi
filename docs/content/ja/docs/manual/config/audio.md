---
title: 音声入出力を構成する
description: 音声合成（TTS）および音声認識（ASR/STT）用の AIRI の設定
---

音声合成 (TTS) は AIRI のテキスト応答を読み上げます。音声認識 (ASR/STT) はマイクの音をテキストに変換します。この 2 つは独立して設定できます。TTS のみを設定すると AIRI の発話も許可され、ASR のみを設定すると音声入力も使用できるようになります。

## 音声合成 (TTS) を構成する

1. **[設定] → [サービス プロバイダー] → [音声合成]** を開き、サービス プロバイダーを選択して資格情報を入力します。
2. **[設定] → [音声]** を開き、構成されているサービス プロバイダー、モデル、およびトーンを選択します。
3. スピーキングページにテストテキストを入力し、再生します。音が聞こえたら、設定が成功したことを意味します。

サービス プロバイダーが OpenAI 音声インターフェイスと互換性がある場合は、[OpenAI 互換 API (TTS)](./providers/speech/openai.md) を参照してください。 OpenRouter の音声インターフェースを使用する場合は、「OpenRouter (TTS)」(./providers/speech/openrouter.md) を参照してください。

## 音声認識 (ASR/STT) を構成する

1. **[設定] → [サービス プロバイダー] → [音声認識]** を開き、サービス プロバイダーを選択して資格情報を入力します。
2. **[設定] → [聴覚]** を開き、設定されているサービス プロバイダーとモデルを選択します。
3. 正しいマイクを選択し、テストを開始して短い文を発話します。
4. 認識結果領域にテキストが正しく表示されることを確認します。

Alibaba Cloud のリアルタイム識別については、[Alibaba Cloud NLS](./providers/transcription/aliyun.md) を参照してください。 OpenAI トランスクリプション インターフェイスと互換性のあるサービスについては、[OpenAI 互換 API (ASR/STT)](./providers/transcription/openai.md) を参照してください。

## よくある質問

### TTS 音が出ない

音声プロバイダー、モデル、トーンが選択されていることを確認し、システムの出力デバイスと音量を確認してください。テストエリアでサービスプロバイダーエラーが表示される場合は、まずAPIキー、バランス、モデルが音声合成をサポートしているかどうかを確認してください。

### ASR テキスト結果がありません

AIRI がマイクに対する許可を持っていること、およびヒアリング ページで正しい入力デバイスが選択されていることを確認します。リアルタイム認識サービスの場合、ネットワークの停止やブラウザ/システムのマイク許可が取り消されると、結果が空になる可能性があります。

### 言語または音色の認識が正しくない

サービスプロバイダーがターゲット言語でサポートしているモデルまたはサウンドを選択します。音声認識の言語、地域、およびモデルの設定は、サービス プロバイダーによって実際に有効にされている機能と一致している必要があります。

## 次のステップ

API キー、ベース URL、検証プロセスの意味を理解するには、[共通設定​​手順](./common.md) をお読みください。サービスプロバイダーの設定方法については、サイドバー「サービスプロバイダー → 音声合成/音声認識」を展開し、目的のプロバイダーを選択します。
